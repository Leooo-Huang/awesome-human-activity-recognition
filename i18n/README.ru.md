Курируемый каталог из **42** наборов данных по активности и движению человека, охватывающий компьютерное зрение, носимые датчики, скелетную запись, мультимодальные данные и многое другое. Каждая карточка набора данных содержит информацию о модальностях, лицензии, эталонных задачах, репрезентативных базовых моделях, экосистеме инструментов и основных публикациях, что позволяет быстро приступить к моделированию или сравнительной оценке.

- Предназначен для исследователей машинного обучения, продуктовых команд и лабораторий двигательных наук.
- Представлены как новейшие наборы данных, так и классические, на основе которых построены современные бенчмарки.
- Включает рабочий процесс для вклада, инструменты валидации и дорожную карту для совместного сопровождения сообществом.

> Цель: стать авторитетным GitHub-ресурсом по наборам данных для распознавания человеческой активности, с глубиной Papers with Code, но с фокусом на «наборы данных в первую очередь».

## Зачем нужен этот репозиторий
- **Навигация по задачам:** Ищите наборы данных по задаче или модальности, а не просто по алфавиту.
- **Исследовательский контекст:** Каждая карточка связана с основными публикациями, бенчмарками и новейшими работами.
- **Рабочие инструменты:** Генератор каталога (экспорт JSON/CSV), проверка ссылок, ASCII-нормализация.
- **Гарантия качества:** Шаблоны для вкладов, контрольные списки и автоматизированный CI для надёжного и достоверного каталога.

## Как пользоваться каталогом
1. **Найти набор данных:** Сначала обратитесь к таблице классификации выше или просмотрите каталог [`datasets/`](../datasets/).
2. **Прочитать карточку набора данных:** Перед загрузкой ознакомьтесь с лицензией, задачами, протоколом бенчмарка и особенностями данных.
3. **Быстрое прототипирование:** Используйте ссылки на инструменты и экосистему в каждой карточке для подключения загрузчиков данных PyTorch / TensorFlow.
4. **Сгенерировать экспорт каталога:** Выполните `python tools/catalog_builder.py --format csv --output catalog.csv` для получения машиночитаемых метаданных.
5. **Оставаться в курсе:** Подпишитесь (Watch) на репозиторий или следите за Discussions; мы подводим итоги по новым наборам данных и достижениям SOTA при каждом релизе.

## Как внести вклад
- Прочитайте [`CONTRIBUTING.md`](../CONTRIBUTING.md) для получения шаблона карточки, критериев рецензирования и стандартов оформления.
- Используйте Issues для запроса нового набора данных или сообщения о нерабочей ссылке. Мы используем метки по модальностям (напр., `modality:wearable`).
- Pull Request проходят проверку качества данных и автоматическую валидацию ссылок; срок рассмотрения — 5–7 дней.
